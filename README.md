1. Цель приложения
Мобильное приложение для Android, которое позволяет пользователю выбрать баскетбольный матч (любые лиги, кроме NBA по желанию), ввести линию тотала от букмекера и получить прогноз: «Тотал больше» или «Тотал меньше» на основе анализа последних 10 игр каждой команды и последних 5 личных встреч. Все ранее рассчитанные матчи сохраняются в историю с возможностью просмотра деталей.

3.2. Получение последних игр команды

GET /games?h2h={team1_id}-{team2_id}&limit=5

Форматы ответов описаны в предыдущей версии ТЗ.

4. Логика работы приложения (пошагово)
4.1. SplashScreen
Отображает логотип и индикатор загрузки.

Загружает матчи на сегодня и завтра (два запроса).

Сохраняет успешно загруженные матчи в локальный кэш (SharedPreferences/DataStore) вместе с временной меткой.

При отсутствии интернета или ошибке – пытается загрузить кэш (если он не старше 6 часов).

После загрузки (из сети или кэша) переходит на MainActivity, передавая список матчей.

4.2. MainActivity (главный экран)
Элементы:

BottomNavigationView с двумя пунктами: «Матчи» и «История» (по умолчанию открыт «Матчи»).

В режиме «Матчи»:

Спиннер выбора лиги (уникальные league.name из загруженных матчей).

Спиннер выбора матча (фильтрация по выбранной лиге).

Поле ввода линии букмекера (число с плавающей точкой).

Кнопка «Рассчитать» (активна только когда выбран матч и введена линия).

(Опционально) карточка с информацией о выбранном матче.

В режиме «История» – см. п. 4.4.

4.3. Расчёт прогноза и отображение результата
Действия при нажатии «Рассчитать»:

Параллельно выполнить три запроса к API:
GET /games?team={home_id}&limit=10
GET /games?team={away_id}&limit=10
GET /games?h2h={home_id}-{away_id}&limit=5
Вычислить home_avg, away_avg, h2h_avg.
predicted_total = (home_avg * 0.3) + (away_avg * 0.3) + (h2h_avg * 0.4)
Вердикт: "БОЛЬШЕ" если predicted_total > line, иначе "МЕНЬШЕ".
Запас = |predicted_total - line|.
Показать результат в отдельном экране (ResultActivity) или диалоге.
После отображения результата автоматически сохранить расчёт в базу данных истории (см. п. 4.4).
4.4. История расчётов (HistoryFragment / Activity)
Хранение: Room Database (таблица calculations_history).

Схема записи:

id (автоинкремент)

match_id (из API)

league_name

home_team_name, home_team_abbr

away_team_name, away_team_abbr

match_date (дата матча)

calculation_timestamp (Unix time, когда был сделан расчёт)

home_avg_points, away_avg_points, h2h_avg_total

predicted_total

bookmaker_line

verdict ("БОЛЬШЕ" или "МЕНЬШЕ")

margin (запас в очках)

Отображение:

Список элементов (RecyclerView), отсортированный по calculation_timestamp DESC (сначала новые).

Каждый элемент списка показывает:
Лига: Название лиги
Хозяева (аббр) vs Гости (аббр) – дата матча
Вердикт: БОЛЬШЕ/МЕНЬШЕ (цветной индикатор)

При клике на элемент – раскрывается дополнительная информация (например, через ExpandableListView или отдельный диалог) с полными деталями расчёта:

Прогноз системы, линия БК, запас

Средние очки хозяев/гостей (с указанием количества игр)

Средний тотал H2H

Дата расчёта (в человекочитаемом виде)

Долгое нажатие – показывает контекстное меню: «Удалить» (одну запись).

Кнопка (или пункт меню) «Очистить историю» – удаляет все записи с подтверждением.

Навигация: Доступ через BottomNavigationView на MainActivity (переключение между вкладками «Матчи» и «История»).

4.5. Дополнительные требования
При первом запуске истории нет.

При повторном расчёте того же матча создаётся новая запись (все расчёты сохраняются).

Если пользователь вышел из приложения и зашёл снова, история сохраняется.

Возможность поделиться результатом расчёта (кнопка «Поделиться» на экране результата) – опционально.

5. Кэширование и локальное хранение
Список матчей (на 2 дня): SharedPreferences/DataStore (6 часов).

История расчётов: Room Database (неограниченно).

Статистика команд (средние очки, H2H): не кэшируется на диск, но можно кэшировать в памяти на время сессии.

6. Обработка ошибок и лимитов
Сетевые ошибки, ошибки API, отсутствие данных – обрабатываются и показываются пользователю.

При превышении лимита API (код 429) – показать сообщение и предложить повторить позже.

При сохранении в историю – ошибки базы данных не должны влиять на отображение результата (логировать, но не показывать пользователю, если запись не сохранилась).

7. Технические требования к Android
Минимальный SDK: 24 (Android 7.0)

Язык: Kotlin

Сетевой слой: Retrofit 2 + OkHttp 3 + Gson

Асинхронность: Kotlin Coroutines (ViewModel + LiveData/Flow)

Кэш матчей: DataStore (Preferences)

История: Room (SQLite)

UI: Material Design, ViewBinding, RecyclerView, BottomNavigationView, Spinner, EditText, Button, ProgressBar.

Навигация: Activity для Splash, Main (с фрагментами) или отдельные Activity для Main и History, ResultActivity.
